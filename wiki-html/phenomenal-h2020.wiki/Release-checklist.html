Before a new release

  • Prepare release document
    • Planning
    • Fix dates for: Alpha, Beta, RC, and actual release
    • Define scope of additions/changes included in the release
    • Setup meetings weekly 30 minute progress meeting with core team
  • Prepare release announcement
  • Assign roles/responsibilities
    • Release manager, infra lead, services (tools/containers/workflows) lead, portal (UI) lead, and user representatives.
